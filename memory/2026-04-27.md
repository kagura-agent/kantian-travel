**Approach:** Clone repo → read architecture (4-layer chain: MCP→bridge→extension→CDP) → trace data flow through source → compare with OpenClaw's Playwright-based approach.

**Direction impact:** Confirmed that headless-first is the right default for OpenClaw (server agents need it). But if OpenClaw ever adds a desktop companion mode, Chrome-reuse via extension+NM is the architecture to follow.

## 📚 Study Loop (18:45) — Apply: Jaccard Clustering for beliefs-candidates

**模式**: apply → note → cascade(skip) → reflect → done

### 应用内容
- **来源**: agentic-stack wiki 笔记的待办 "研究 Jaccard 聚类是否适合我们的 gradient 自动聚合"
- **产出**: `tools/gradient-cluster.py` — 纯 Python 脚本，无外部依赖

### 成功分析
- goal: 给 beliefs-candidates.md 的 110 条 gradient 做自动相似度聚类，找重复/可合并的
- approach: 从 agentic-stack 的 Jaccard 聚类出发，发现纯词法对 CJK 不行（Jaccard=0.06 for clearly related entries），改用 concept vocabulary mapping
- key_decisions: 放弃 CJK bigram 方案（token 太碎），改用 concept normalization（表层形式→规范化概念）+ Latin keywords
- pattern: **混合语言文本匹配需要概念抽象层**——agentic-stack 的 Jaccard 适用于纯英文，CJK+EN 需要先 normalize 到共同概念空间
- applies_when: 对任何 CJK/English 混合文档做相似度分析时

### 失败分析
- failure_point: 第一版 CJK bigram tokenizer 产生太多 unique token，Jaccard 极低
- flawed_reasoning: 假设 character-level bigrams 足够捕捉语义相似度
- what_should_have_been_done: 一开始就用 concept vocabulary（参考 agentic-stack 的 conditions 2x 权重思路）
- prevention_principle: 对非空格分词语言，词法匹配前先考虑是否需要概念归一化

### 验证结果
- threshold 0.35: 6 clusters（含 30 entry mega-cluster → 需更高 threshold）
- threshold 0.45: 4 clusters，更精准，每个 cluster 内确实是同一主题
- 真正的聚类发现：skip-own-tools (2 entries)、协作边界 (2 entries)、项目建制 (2 entries by pattern tag)
- 工具可用性：✅ 零依赖，直接 python3 运行

### 方向影响
- 验证了 agentic-stack 的核心洞察："聚类不需要 embedding"——对，但需要 domain-specific 的 concept vocabulary
- 新发现：beliefs-candidates 的 52/110 有 pattern tag，58 没有 → untagged entries 是聚类工具的主要目标用户
