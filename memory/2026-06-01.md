## Study Apply (09:45) — Fix weekly-eval.sh (6-week-old bug)

**Applied**: 修复 `scripts/weekly-eval.sh` line 72 integer comparison error
**Root Cause**: `grep -c` 无匹配时输出 "0" 到 stdout 然后 exit 1，`|| echo 0` 追加第二个 "0"，变成 "0\n0"，`[ "$VAR" -gt 0 ]` 解析失败
**Fix**: `|| echo 0` → `|| true` + `${VAR:-0}` 默认值。同时扩展 UPGRADED 匹配模式加入 `已毕业/graduated`（beliefs-candidates 实际用的术语）
**Before**: 每次运行报 `integer expression expected` 错误，UPGRADED 始终显示 "0\n0"，6 周未修
**After**: 无错误运行，UPGRADED 正确显示 12（匹配实际毕业数）
**Verification**: `bash scripts/weekly-eval.sh` → 无 error，报告 UPGRADED=12 ✅
**Commit**: 975da60 pushed to kagura-agent/dna
**Lesson**: `grep -c || echo 0` 是 bash 经典陷阱 — grep -c 总输出到 stdout（即使 0），exit code 才表示有无匹配。用 `|| true` 更安全
