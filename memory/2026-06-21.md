### Reflect (10:25 — study apply: stale-workloop-recovery)

**成功分析**:
- goal: Convert stale-workloop-recovery from behavioral gradient → structural gate
- approach: Create dedicated script (stale-pr-check.sh) + integrate into workloop.yaml as Branch 3
- key_decisions: (1) Three exit codes (10/11/0) rather than binary — fix mode (11) prevents unnecessary redo even when CI fails. (2) Placed before competing-pr-check — stale recovery is more urgent than competition detection.

- what_should_have_been_done: Use `|| true` pattern from the start (known bash idiom for grep in strict mode)
- prevention_principle: When writing bash with `set -euo pipefail`, always use `|| true` for grep -c, never `|| echo "0"`

**Gradient 自检**: No new gradient needed — the `|| true` pattern is already well-known. This round was clean otherwise (90 min from start to verified commit). The pattern of "test against real PRs before committing" worked well (caught the grep bug immediately).
