1. **`goal_state.py`** — Pure data layer. Session metadata stores goal as JSON blob (`{status, objective, ui_summary, started_at}`). Handles legacy key migration. Two projections:
   - `goal_state_runtime_lines()` → injected into Runtime Context block each turn (max 4000 chars) — survives compaction
   - `goal_state_ws_blob()` → WebSocket events for WebUI (max 600 chars)

2. **`long_task.py`** — Two tools:
   - `long_task(goal, ui_summary?)` — registers objective on session metadata. Rejects if goal already active ("finish or complete_goal first"). Mandates reading the `long-goal` skill first for idempotent, compaction-safe wording
   - `complete_goal(recap?)` — marks goal completed with timestamp and recap. Handles success, cancellation, and redirection honestly

3. **Loop integration** — `goal_state_runtime_lines()` injected via `supplemental_lines` param on `_build_runtime_context()`. Goal state broadcast on turn-end via `_turn_end` metadata. Turn latency tracking (`turn_wall_started_at → latency_ms` on last assistant message)

**Key design decisions:**
- **No sub-agent orchestrator** — explicit design choice. Work proceeds in ordinary turns with normal tools. The goal just provides persistent context across compaction
- **Compaction-safe by design** — goal text re-injected every turn from session metadata, not from message history. Compaction can't hide it
- **Single-goal constraint** — one active goal per session. Must complete/cancel before starting another
- **Idempotent goal wording mandated** — tool description requires reading `long-goal` skill for "desired outcomes and acceptance criteria, not brittle step lists that break on retry"

**Relevance to us:**
- OpenClaw's heartbeat/cron tasks have no persistent objective tracking — each run starts fresh from HEARTBEAT.md
- Our FlowForge workflow state partially serves this role (current node = persistent objective) but it's not injected into agent context automatically
- The "supplemental_lines in runtime context" pattern is elegant — could inject FlowForge state or TODO priorities into every turn
- The compaction-safety principle is important: anything that must survive context window rotation belongs in session metadata, not message history
