## Study Apply (09:32) — Gradient Scan Signal Extraction

**Source insight**: Elephant Agent PR#43 Skill Optimization Pipeline — automated signal extraction from historical episodes, applied to our beliefs-candidates pipeline.

**Problem**: 16 beliefs-candidates stuck at count=1. Pipeline has no automated way to detect when patterns recur across sessions. Relies entirely on manual observation (nudge/reflect), which isn't producing enough gradients.

**Created**: `tools/gradient-scan.sh`
- Extracts pattern tags from beliefs-candidates.md
- Searches recent memory files for keyword matches (configurable --days window)
- Skips already-logged dates for each pattern (avoids double-counting)
- Reports hits with evidence dates
- Integrated into review.yaml beliefs_graduation as step 0

**Iteration**: First version had broad keywords → 218 false positives (15 patterns matched). Tightened to behavior-specific keywords → 23 genuine hits (3 patterns). Lesson: pattern keywords should match the error/violation, not the domain context.

**Immediate impact**:

- Both now clearly pass V1 (≥3 independent occurrences) — were invisible at count=1
