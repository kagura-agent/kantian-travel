**Applied**: Fixed 3x-gradient tool-friction-sed-bug in `study/tracking-update.sh`:
1. **awk -v backslash interpretation** (main bug): `awk -v new="$var"` interprets `\n`→newline, `\t`→tab in content. Fixed by using `ENVIRON["_FF_NEW_LINE"]` which preserves literal backslashes.
2. **echo → printf**: Replaced `echo "$var"` with `printf '%s\n' "$var"` in sed pipelines for portability (some echo implementations interpret backslash sequences).

**What's different**: Notes with backslashes (e.g., paths, escape sequences) no longer corrupt the markdown table. Previously `\n` in notes would create a literal newline in the table row, breaking subsequent parsing.
